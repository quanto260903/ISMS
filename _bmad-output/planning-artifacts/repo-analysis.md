# Repository Analysis

## Overview

This document captures the current structure and key components of the ISMS repository.
It focuses on:
- Backend source code (ISME_BE)
- Frontend source code (ISMS_FE)
- Documentation artifacts (docs/output)

## Repository Layout

### Top-level folders
- `ISME_BE`: .NET backend API project(s)
- `ISMS_FE`: Next.js frontend project
- `docs`: Official (finalized) documentation
- `_bmad-output`: intermediate planning artifacts for AI agent workflows
- `.github`: agent configurations and workflow rules

## Backend (ISME_BE)

### Likely structure (based on conventions and documentation)
- `src/` (expected): Contains core application code
  - `Controllers/` - HTTP API controllers
  - `Services/` - business logic layer
  - `Repositories/` - data access layer
  - `Models/` - domain/data models
  - `Config/` - configuration (e.g., appsettings)
- `tests/` (expected): automated tests (unit/integration)
- `.csproj` and tooling files: project configuration, dependencies

### Key backend responsibilities
- Expose RESTful APIs for frontend clients
- Implement authentication / authorization (JWT-based)
- Manage database access via Entity Framework Core or equivalent
- Provide transaction and reporting services for inventory, sales, purchases, payments
- Support role-based access and activity logging

## Frontend (ISMS_FE)

### Likely structure (based on conventions and documentation)
- `src/` (expected): main source code
  - `components/` - reusable UI components
  - `pages/` or `app/` - Next.js routes
  - `services/` - API client utilities
  - `hooks/` - shared logic hooks
  - `utils/` - helper utilities
  - `store/` - state management (Redux/Zustand)
  - `types/` - TypeScript types and interfaces
- `public/` - static assets (images, icons, etc.)
- `package.json`, `tsconfig.json`, `next.config.js` etc.

### Key frontend responsibilities
- Provide user interfaces for inventory, sales, purchases, reporting, and administration
- Manage authentication flow and local session state
- Consume backend REST APIs and render responses
- Implement voice search / AI features (via browser speech APIs)
